Core Semantic Elements

Each card explains what the element is for and where it belongs.

<header>

  • Intro content (logo/title)
  • Can appear inside sections/articles
  • Often paired with navigation

<nav>

  • Primary navigation links
  • Not for every link list
  • Should be keyboard-friendly

<main>

  • Main content of the page
  • Only one per page
  • Skip link should land here

<section>

  • Groups related content by theme
  • Usually has a heading
  • Helps page structure

<article>

  • Self-contained content
  • Blog post, card, news item
  • Can stand alone

<aside>

  • Secondary/related content
  • Sidebar, tips, related links
  • Not the main story

<footer>

  • Footer for page or section
  • Metadata, links, copyright
  • Can appear more than once

Form Example (Labels)

Accessibility Essentials

  • Use <label for> with inputs
  • Meaningful alt text for images
  • Logical headings (h1 → h2 → h3)
  • Keyboard navigation works

Common Mistakes

  • Overusing <div> instead of semantic tags
  • Using multiple <main> elements
  • Skipping heading levels
  • Missing labels on form inputs

Key Takeaways

  • Semantics improves usability and clarity
  • Screen readers rely on structure
  • Maintainable code starts with good HTML
